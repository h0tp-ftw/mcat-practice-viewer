How to build a question set

A question set is a single .json file. Load it into the viewer by dragging it anywhere onto the page, or using Load JSON in the top bar. This page describes exactly what that file should contain.

The file

The file is a JSON array of question objects. The viewer sorts them by their order field before showing them. Each object is one question.

Question object

FieldTypeRequiredWhat it does
idnumberyesA unique id for the question.
ordernumberyesPosition in the set (questions are shown in ascending order).
passage_idnumber or nulloptionalGive every question that shares one passage the same value, so they are grouped. Use null for a standalone (discrete) question.
passageHtmlstring or nulloptionalHTML shown in the left passage pane. Use null for a discrete question (the viewer keeps the two-pane layout and notes that the item is independent of a passage).
question_contentstringyesHTML for the question stem.
answer_choicesarrayyesThe list of choice objects (see below). MCAT items use four.
correct_answerstringoptionalThe correct letter, e.g. "C". Display only — correctness is taken from the correct flags, not this field.
answer_contentstringoptionalHTML explanation, shown when the answer is revealed.

Choice object

FieldTypeRequiredWhat it does
idnumberyesUnique within the question.
sort_ordernumberyes0-based. Becomes the letter: 0→A, 1→B, 2→C, 3→D.
choice_contentstringyesHTML for the choice text.
correctbooleanyesExactly one choice per question must be true.

Good to know

  • Passages. Questions with the same passage_id are treated as one passage and numbered in the order they first appear.
  • Passage titles. If your passageHtml already begins with a header like Passage 1 (3 questions), the viewer hides its own generated title to avoid a duplicate. If it doesn't, the viewer adds one for you.
  • Discrete questions. Set both passage_id and passageHtml to null.
  • HTML is rendered. The passage, stem, choices, and explanation are inserted as HTML, so use tags like <p>, <strong>, <em>, tables, etc. Only load content you trust.
  • Images. Reference a local file with a relative path, e.g. <img src="./images/figure1.png">. Remote http(s) URLs only work while online.
  • Keep a local copy. The viewer auto-loads a file named content.json if it sits next to the app, so you can drop your set in as content.json to have it open on startup.

Complete example

A minimal, working set: one passage with two questions, plus one discrete question. Copy it, replace the placeholder text, and load it into the viewer.

[
  {
    "id": 1,
    "order": 1,
    "passage_id": 100,
    "passageHtml": "<p><strong>Passage 1 (2 questions)</strong></p><p>Your passage text goes here…</p>",
    "question_content": "<p>What is the author's main point?</p>",
    "answer_choices": [
      { "id": 11, "sort_order": 0, "choice_content": "<p>First choice</p>",  "correct": false },
      { "id": 12, "sort_order": 1, "choice_content": "<p>Second choice</p>", "correct": true  },
      { "id": 13, "sort_order": 2, "choice_content": "<p>Third choice</p>",  "correct": false },
      { "id": 14, "sort_order": 3, "choice_content": "<p>Fourth choice</p>", "correct": false }
    ],
    "correct_answer": "B",
    "answer_content": "<p><strong>The answer is B.</strong> Because…</p>"
  },
  {
    "id": 2,
    "order": 2,
    "passage_id": 100,
    "passageHtml": "<p><strong>Passage 1 (2 questions)</strong></p><p>Your passage text goes here…</p>",
    "question_content": "<p>A second question about the same passage.</p>",
    "answer_choices": [
      { "id": 21, "sort_order": 0, "choice_content": "<p>First choice</p>",  "correct": true  },
      { "id": 22, "sort_order": 1, "choice_content": "<p>Second choice</p>", "correct": false },
      { "id": 23, "sort_order": 2, "choice_content": "<p>Third choice</p>",  "correct": false },
      { "id": 24, "sort_order": 3, "choice_content": "<p>Fourth choice</p>", "correct": false }
    ],
    "correct_answer": "A",
    "answer_content": "<p><strong>The answer is A.</strong></p>"
  },
  {
    "id": 3,
    "order": 3,
    "passage_id": null,
    "passageHtml": null,
    "question_content": "<p>A standalone question that is not based on a passage.</p>",
    "answer_choices": [
      { "id": 31, "sort_order": 0, "choice_content": "<p>First choice</p>",  "correct": false },
      { "id": 32, "sort_order": 1, "choice_content": "<p>Second choice</p>", "correct": false },

    ],
    "correct_answer": "C",
    "answer_content": "<p><strong>The answer is C.</strong></p>"
  }
]
